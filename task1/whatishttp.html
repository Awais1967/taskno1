How is a web page downloaded?

Overview

When you type a URL into your web browser and press Enter, a series of HTTP requests and responses are exchanged between your browser and the web server to download and display the webpage. Let's examine the sequence of these requests using the browser's network console.

Using the Network Console

To understand how a web page is downloaded, follow these steps:

  1. Open your web browser (e.g., Google Chrome).
  2. Press F12 to open the Developer Tools.
  3. Navigate to the Network tab.
  4. Type a URL into the address bar and press Enter to load the page.

You will see a list of network requests made by your browser to load the page. Each row represents a request for a resource, such as an HTML file, CSS file, JavaScript file, or image.

HTTP Requests and Responses

Here is an example trace of the HTTP requests made when loading a simple webpage:

# Resource Type Status Size Time
1 index.html document 200 OK 2.3 KB 100 ms
2 style.css stylesheet 200 OK 1.1 KB 50 ms
3 script.js script 200 OK 1.4 KB 75 ms
4 logo.png image 200 OK 5.2 KB 120 ms

Annotated Explanation

Let's analyze the sequence of HTTP requests:

1. index.html

This is the main HTML document that defines the structure of the webpage. The browser requests this file first. Status 200 OK indicates the request was successful.

2. style.css

This stylesheet file is used to style the HTML document. The browser requests this file after parsing the HTML and finding the link to the CSS file.

3. script.js

This JavaScript file contains the script that adds interactivity to the webpage. The browser requests this file after finding the script tag in the HTML.

4. logo.png

This image file is used to display the logo on the webpage. The browser requests this file when it encounters the image tag in the HTML.
