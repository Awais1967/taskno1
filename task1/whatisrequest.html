What is a Request?

Overview of HTTP Requests

An HTTP request is a message sent by the client to a server in order to fetch resources or perform operations. Each HTTP request consists of several parts: the request line, headers, and an optional body.

Request Line

The request line is the first line in an HTTP request message. It contains the method, the URI (Uniform Resource Identifier), and the HTTP version. For example: 

GET /index.html HTTP/1.1

In this example, GET is the method, /index.html is the URI, and HTTP/1.1 is the HTTP version.

Request Headers

Headers provide additional information about the request. They are key-value pairs sent in the request message. Common headers include:

  • Host: Specifies the domain name of the server (e.g., Host: www.example.com).
  • User-Agent: Provides information about the client software making the request (e.g., User-Agent: Mozilla/5.0).
  • Accept: Informs the server about the types of data the client can process (e.g., Accept: text/html).

Request Body

The body of a request is optional and is used to send data to the server. It is typically used with methods like POST and PUT. For example, a POST request to submit a form might include data in the body: 

POST /submit-form HTTP/1.1
Host: www.example.com
Content-Type: application/x-www-form-urlencoded

name=John&age=30

Common HTTP Methods

HTTP defines several methods, each serving a different purpose:

  • GET: Requests data from a specified resource. It is the most common method used for retrieving data.
  • POST: Submits data to be processed to a specified resource. It is often used for submitting form data.
  • PUT: Updates a current resource with new data.
  • DELETE: Removes the specified resource.
  • HEAD: Similar to GET, but it only requests the headers of a resource, not the body.

Common HTTP Status Codes

Status codes are issued by a server in response to an HTTP request. They indicate whether the request has been successfully completed. Some common status codes include:

  • 200 OK: The request was successful.
  • 404 Not Found: The requested resource could not be found.
  • 500 Internal Server Error: The server encountered an error and could not complete the request.
  • 301 Moved Permanently: The resource has been permanently moved to a new location.
  • 302 Found: The resource has been temporarily moved to a new location.
